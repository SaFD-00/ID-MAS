# 문서 업데이트 계획: README.md & ARCHITECTURE.md

## 개요

ID-MAS 프로젝트에 많은 리팩토링이 있었습니다. 이 계획은 README.md와 ARCHITECTURE.md를 현재 코드베이스에 맞게 업데이트하는 작업을 정의합니다.

## 주요 변경 사항 분석

### 1. LangGraph 기반 3-Phase Pipeline 추가 (신규)

**위치**: `learning_loop/graph/`

| 파일 | 설명 |
|------|------|
| `__init__.py` | 모듈 export (IDMASState, QuestionResult, IDMASGraphRunner) |
| `state.py` | TypedDict 기반 상태 스키마 (IDMASState, QuestionResult, DesignResult, SFTCase) |
| `nodes.py` | 파이프라인 노드 함수들 (process_question_phase1, generate_coaching_db 등) |
| `graph.py` | StateGraph 구성 및 IDMASGraphRunner 클래스 |

**핵심 기능**:
- StateGraph 기반 워크플로우 관리
- 조건부 라우팅 (should_continue_phase1, should_run_phase2, should_run_phase3)
- MemorySaver 기반 체크포인트
- 파일 기반 Resume 지원 (logs 파일에서 복원)

### 2. 모델 래퍼 통합

**변경 전**:
- `models/gpt_wrapper.py`
- `models/qwen_wrapper.py`

**변경 후**:
- `models/llm_wrapper.py` (LLMWrapper 클래스로 통합)
  - OpenAI API 및 LLaMA-Factory(vLLM) 호환
  - `GPTWrapper = LLMWrapper` (backward compatibility)

### 3. Design Modules 파일명 변경

| 변경 전 | 변경 후 |
|---------|---------|
| `step2_analysis.py` | `analysis.py` |
| `step4_objectives.py` | `objectives.py` |
| `step5_rubric.py` | `rubric.py` |
| `step5_test.py` | `test.py` |

### 4. Config 모듈 구조 변경

`config/` 서브모듈로 분리:
- `api.py`: API 키 및 인증 (PROJECT_ROOT, OPENAI_API_KEY, HF_TOKEN)
- `models.py`: Teacher/Student 모델 설정
- `sft.py`: SFT 모델 매핑
- `domains.py`: 도메인 및 데이터셋 설정
- `paths.py`: 디렉토리 경로 헬퍼

**새로운 환경변수**: `LLAMA_FACTORY_BASE_URL` (기본값: `http://localhost:2000/v1`)

### 5. 삭제된 파일들

- `tests/` 디렉토리 전체 (test_*.py 파일들)
- `pytest.ini`
- `migrate_data_structure.py`
- `update_train_prompts.py`
- `test_system.py`
- `scripts/sort_learning_data.py`

#### 2.6 그래프 흐름도 업데이트

현재 `graph.py` 기반 정확한 흐름:

```
START → phase1 → advance → [조건부 라우팅]
                          ├─ (더 많은 문제) → phase1
                          └─ (Phase 1 완료) → coaching_db
                                              ↓
                                        [조건부 라우팅]
                                        ├─ (오답 있음) → phase2
                                        └─ (모두 정답) → finalize
                                              ↓
                                        [조건부 라우팅]
                                        ├─ (여전히 오답) → phase3
                                        └─ (모두 수정됨) → finalize
                                                           ↓
                                                         END
```
